1. Background & Business Context

Veer & Sant Real Estate LLC is a licensed Dubai real estate brokerage. We are building the
V&S Deal Exchange -- a private, broker-to-broker deal flow platform for the Dubai market.

The problem we are solving: Dubai's off-market property deal flow currently happens via
WhatsApp groups and informal broker networks. This is unstructured, unverified, and chaotic.
Serious deals get lost. Stale listings circulate. There is no quality control.

Our solution: A gated, verified broker network where only DLD-licensed brokers can access and
post. Listings are moderated. Buyer requirements are live. Co-broke terms are explicit. The
platform is premium, selective, and deal-focused -- not a mass-market consumer portal.

Competitor context

The closest comparable in the market is offmarketdxb.com -- a simple custom web app that
functions essentially as a WhatsApp lead generator with listings. It has almost no moat. Our
platform is positioned as a step above: verified network, structured deal flow, admin-controlled
quality.

What we are NOT building

    � A consumer-facing property search site (not Property Finder, not Bayut)
    � A CRM or full agency management system
    � A mobile app (web only for this stage)
    � Automated DLD verification (manual admin verification in this MVP)

2. Project Scope -- MVP (Stage 1)

The MVP must deliver a fully functional web application with the following eight core screens
and their associated logic. A Figma-equivalent interactive HTML prototype is available to
accompany this brief.

2.1 Public screens (unauthenticated)

Screen 1: Homepage / Hero
    � Navigation bar with V&S branding and logo
    � Hero section with headline, value proposition, key platform stats (listings, brokers, deal
        flow)
    � Preview cards: sample listings and buyer requirements (non-interactive, illustrative)
    � "How It Works" section -- 4-step explainer
    � Primary CTA: "Request Broker Access"  leads to Apply screen
    � "Broker Login" link for approved brokers

Screen 2: Apply to Join
    � Left panel: value proposition, platform perks, "Already approved? Login" link
    � Right panel: broker application form (see Section 4 for full field list)
    � On submit: confirmation message displayed, application sent to admin queue
    � No automated email needed in MVP -- admin reviews manually

2.2 Authenticated broker screens (post-login)

All authenticated screens share a persistent sidebar navigation and top navbar. The sidebar
collapses the public nav and shows broker-specific links.

Screen 3: Broker Dashboard
    � Summary metrics: my active listings, new enquiries, buyer requirements matching my
        areas
    � My listings preview (last 3, with status badges: Live / Pending / Expiring)
    � Matched buyer requirements feed (filtered to broker's speciality areas)
    � Recent enquiry activity

Screen 4: Browse Listings
    � Left sidebar: filters (deal type, area, bedrooms, price range, property type)
    � Listings feed: each card shows deal type badge, title, price, co-broke %, broker tag, status
    � Sort by: newest, price (high/low), co-broke %
    � Each listing links to Listing Detail screen
    � Enquire / WhatsApp CTA on each card

6. Acceptance Criteria

The following must be met for the project to be considered complete and payment released in
full.

6.1 Functional requirements

    1. All 8 screens specified in Section 2 are built and functional
    2. All 3 user roles (public, broker, admin) work correctly with proper access control
    3. Broker application form submits and appears in admin queue
    4. Admin can approve / reject broker applications
    5. Approved broker can log in and access all broker screens
    6. Broker can post a listing -- it enters pending state
    7. Admin can approve / reject listings
    8. Approved listing appears in Browse Listings feed with correct filters working
    9. Broker can post a buyer requirement
    10. Enquiry modal sends enquiry and notifies listing broker via email
    11. Listing renewal prompt triggers after 7 days of no reconfirmation
    12. Admin can suspend a broker account
    13. Admin can export leads as CSV

6.2 Non-functional requirements

    � Responsive design -- must work cleanly on desktop (1280px+) and mobile (375px+)
    � Page load time under 3 seconds on standard broadband
    � Secure authentication -- passwords hashed, sessions managed correctly
    � Row-level security: brokers cannot access other brokers' private notes or unapproved

        data
    � All form inputs validated client-side and server-side
    � Images optimised on upload (max 2MB per image stored)
    � SSL / HTTPS enforced on all pages
    � No broken links or console errors in production build

6.3 Deliverables

    � Fully deployed live URL (production environment)
    � Source code in a private GitHub repository -- ownership transferred to Veer & Sant
    � Admin login credentials
    � Environment variables documented in a README
    � 2 weeks of bug-fix support post-launch (response within 48 hours)
